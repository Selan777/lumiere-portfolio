About the project

What this landing page solves

Interface states

Not just beautiful screens, but real UX scenarios

For a portfolio case, it is important to show that the design is thought through beyond the first screen and accounts for real user situations.

Empty cart

The user gets a clear message and a quick way back to the catalog without reaching a dead end in the flow.

No products found

If the filters return no results, the interface explains the situation and suggests adjusting the selection.

Successful order

After the order is confirmed, the user sees a calm success state with guidance on the next step.

Mobile UX

Filters stay out of the way while browsing the catalog

On mobile, filters open in a separate drawer layer. This keeps the focus on product cards without overloading the first screen with interface controls.

  • drawer-based filters
  • single-column catalog grid
  • large tap targets
